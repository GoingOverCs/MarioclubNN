HTML/CSS Crash Course

CSS Basics

  • Inline Elements don't take up more room than their content needs. Examples are span, img, strong, em, a and more
  • Block Elements take up the whole width of the page regardless of content, such as, p. div, the headlines (h1,2 etc.) ul, li and more. Always start on a new line
  • Block line elements can have margins all around, but inline elements can only have margins to the left and to the right of the element,but padding goes all the way around for both and padding is inside the margin
  • Use inline-block to get the best of both worlds because it puts the elements horizontally, but also gives you margins all around

CSS Classes & Selectors

  • If you want to give an element multiple classes just add another word in the class with a space inbetween
  • You can only use one id per element
  • element[attribute] - this so you can targer a certain element with a certain attribute
  • a[href*="amazon"] {} - delivers CSS to any link with the word amazon in it
  • a[href$=".com"] {} - delivers CSS to anything that ends in .com

The Cascade

Inheritence

  • HTML elements can inherit CSS properties that are applied to their parents
  • Inheritence mainly works for text based attributes not things like padding and margin, more things like color and font weight

HTML5 Semantics

  • main - for the main content of a webpage, unique to that page
  • section - Defines a certain section of a webpage (e.g. blog list, contact info)
  • article - Defines a bit of content which makes up an article(e.g. a blog post)
  • aside - Defines some content related to something else (e.g. similar blogs)
  • For the header of a website - contains the nav, title etc
  • footer - for the footer of a website
  • When making a list it doesnt have to be a traditonal list it can be a list of sites,pictures, etc.

CSS Layout and Position

  • Position can have 5 values
    • static - default value, keeps element in normal document flow of the webpage
    • Relative - you can shift the element relative to its original place on the page
    • fixed - positions an element realtive to the viewpoint. the viewpoint is just the place where we see what's on our browser
    • absolute - allows us to position things absolutely relative to its closest parent which has alos been given a position property that is not static. when you position something absolutely you take it out of normal document flow
    • sticky - a mixture of static and fixed. The element stays static until a certain point that you set and then it becomes fixed
  • em multiplies the size of the parent by the value next to the em
  • z index tells the computer the importance of an element and every element starts off with z index of 0. The higher the z-index number the more important it is for the computer to show
  • white-space:nowrap makes it so when you use inline block your text/images do not wrap to the next line
  • When you use margin auto it automatically makes an even about of space on the sides you choose
  • If you make your left and right margin auto it essentially centers your content Ex: margin: 80px auto; (80px is for top and bottom, auto is fot left and right)
  • box-sizing: border-box; makes padding inclusive to when 100% max width is set, meaning the page doesn't take padding into account when the page is resized to a smaller than regular width size

Pseudo Class & Elements

  • Pseudo elements are keywords that you add to selectors to affect elements when they are in a certain state such as hover, focus, first child of a parent element, etc.
  • :hover class activates when you hover over an element
  • :focus activates when a element is actively being used
  • :first child affects only the first child of the parent element
  • :valid affects the element when the proper value is reached like for example in an email input, when a valid email is put in (meaning it has text and an @ sign, the psuedo class gets activated
  • ::first-line affects only the first line of an element
  • ::selection changes the effect of when something is highlighted, like highlighting text
  • ::after puts things after a certain element while ::before puts things before a certain element

Intro to Media Queries/Responsive design

  • Media Queries tell the browser how to style an element at particular viewport dimensions
  • Viewport meta tag tells the browser what width the viewport should be and if we should zoom in or out
  • Responsive images are images that a loaded at a smaller file size for mobile devices
  • when planning to make a website responsive you want to have a strategy and clear process put into place. One popular way is a mobile first design path which means designing your website for mobile and then adapting it for larger screens. This is the way to go so you only use the most essential information and style because you do not have a lot of space
  • the viewport meta tag goes into the head of your HTML file
